Form Backend Setup Guide

This guide walks you through connecting your 4 form destinations. Total time: ~25 minutes. All services are free.

What you're setting up:
  • 📧 Gmail SMTP — sends team notifications + auto-replies
  • 📊 Google Sheets — stores every form submission
  • 💬 Microsoft Teams — instant card notification per submission
  • 🔑 Vercel Environment Variables — connects it all securely

Step 1 — Gmail App Password (~3 min)

Your form sends email via Gmail SMTP. You need an App Password — a separate 16-character password just for this app. Your main Google password is never exposed.

⚠️ Gmail 2-Step Verification must be ON before you can create App Passwords. Check at myaccount.google.com/security.
1. Go to myaccount.google.com → Security
2. In the search bar at the top, type App Passwords and click it
3. App name: type Action Insights Forms → click Create
4. Google shows a 16-character password. Copy it now — you won't see it again. It looks like: abcd efgh ijkl mnop

You'll add this as GMAIL_APP_PASSWORD in Vercel in Step 4.

Step 2 — Google Sheets Setup (~8 min)

Every form submission gets logged to a Google Sheet you control. This requires a Service Account — a bot Google identity that can write to your sheet.

2a. Create the Google Sheet

1. Go to sheets.google.com → create a new sheet → name it Action Insights — Form Submissions
2. Add these headers in row 1 (one per column A through J):
Timestamp   First Name   Last Name   Email   Company   Role   Team Size   CRM   Challenge   Form Type
3. Copy the Sheet ID from the URL:
https://docs.google.com/spreadsheets/d/THIS_IS_YOUR_SHEET_ID/edit

2b. Create a Service Account

1. Go to console.cloud.google.com → click the project dropdown at the top → New Project → name it action-insights-forms → Create
2. In the left sidebar: APIs & Services → Library → search Google Sheets API → click it → Enable
3. Left sidebar: IAM & Admin → Service Accounts → + Create Service Account
Name: sheets-writer → Continue → Role: Editor → Done
4. Click the service account email → Keys tab → Add Key → Create new key → JSON → Create
A JSON file downloads. Keep this safe — it's your credentials.
5. Open the JSON file. Find and copy:
• "client_email" → this is your GOOGLE_SERVICE_ACCOUNT_EMAIL
• "private_key" → this is your GOOGLE_PRIVATE_KEY (the long block starting with -----BEGIN RSA PRIVATE KEY-----)

2c. Share the sheet with the service account

1. Back in your Google Sheet → click Share
2. Paste the service account email (looks like sheets-writer@action-insights-forms.iam.gserviceaccount.com) → give it Editor access → Send

Step 3 — Teams Incoming Webhook (~3 min)

1. In Microsoft Teams, go to the channel where you want notifications (e.g. #sales-leads)
2. Click the ··· menu next to the channel name → Connectors
3. Search for Incoming Webhook → click Configure
4. Name: Action Insights → optionally upload a logo → click Create
5. Copy the webhook URL — it looks like:
https://your-org.webhook.office.com/webhookb2/xxxxx@xxxxx/IncomingWebhook/xxxxx

This is your TEAMS_WEBHOOK_URL.

Step 4 — Add Variables to Vercel (~5 min)

Never put secrets in your code. Vercel stores them securely and injects them at runtime.

1. Go to vercel.com → your project → Settings → Environment Variables
2. Add each of these variables (Name + Value), selecting All Environments:
Variable NameValue
GMAIL_USERYour Gmail address, e.g. you@gmail.com
GMAIL_APP_PASSWORDThe 16-char App Password from Step 1 (no spaces)
NOTIFY_EMAILWhere team notifications go, e.g. team@company.com
TEAMS_WEBHOOK_URLThe full webhook URL from Step 3
GOOGLE_SHEET_IDThe Sheet ID from Step 2a
GOOGLE_SERVICE_ACCOUNT_EMAILThe client_email from the JSON file
GOOGLE_PRIVATE_KEYThe entire private_key value from the JSON file. Paste as-is including the -----BEGIN... lines.
3. Click Save after each variable
4. Go to Deployments → click the three dots on your latest deployment → Redeploy (so the new env vars take effect)

Step 5 — Update Your Domain in the Auto-Reply Emails

In api/submit.js, find the three instances of https://YOUR_DOMAIN in the auto-reply HTML and replace with your actual domain, e.g. https://insights.actioncrm.io.

// Find these lines in api/submit.js and update:
href="https://YOUR_DOMAIN/article-1"
href="https://YOUR_DOMAIN/article-2"
href="https://YOUR_DOMAIN/article-3"

Step 6 — Test It

1. Submit the consultation form on your live site with a real email address you can check
2. Within 30 seconds you should see: ✓ Team email in your inbox   ✓ Auto-reply to the submitted address   ✓ Teams card in your channel   ✓ New row in your Google Sheet
3. If something is missing: Vercel Dashboard → your project → Functions tab → click api/submit → view logs to see any error messages

Troubleshooting

SymptomLikely cause
Gmail fails to sendApp Password incorrect, or 2-Step Verification not enabled. Regenerate the App Password.
Google Sheets not loggingService account not shared on the sheet with Editor access (Step 2c). Private key pasted incorrectly — must include the -----BEGIN and -----END lines.
Teams notification not appearingWebhook URL expired or copied incorrectly. Recreate the webhook in Teams Connectors.
Form shows error on submitCheck Vercel Function logs. Usually a missing env variable.
